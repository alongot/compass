## Directory Purposes

**src/:**
- Purpose: All frontend source code
- Contains: React component(s), hooks, utilities, data
- Key files: `main.jsx` (entry), `CompassDemo.jsx` (main UI)

**src/hooks/:**
- Purpose: React hooks for data fetching and state management
- Contains: Supabase query hooks with loading/error states
- Key files: `useDatabase.js` (primary), `useCourses.js` (legacy)

**src/lib/:**
- Purpose: Initialization and error handling for external libraries
- Contains: Supabase client setup, error helpers
- Key files: `supabase.js` (Supabase configuration)

**src/utils/:**
- Purpose: Reusable functions and API wrappers
- Contains: UCSB API wrapper functions
- Key files: `ucsbApi.js` (UCSB API calls)

**src/data/:**
- Purpose: Data assets and database schema
- Contains: JSON datasets, SQL migrations
- Key files: `migrations/001_initial_schema.sql` (database schema)

**scripts/:**
- Purpose: Data engineering and bulk operations
- Contains: Web scrapers, data importers, PDF parsers
- Key files: `import-courses.js`, `import-programs.js`, `transcriptparser.py`

**tests/:**
- Purpose: Unit and integration tests
- Contains: Test files (currently minimal)
- Key files: None currently active

**docs/:**
- Purpose: Project documentation
- Contains: Architecture guides, roadmaps, data dictionary
- Key files: `DATABASE.md`, `ROADMAP.md`, `DEMO_ROADMAP.md`

## Naming Conventions

**Files:**
- Components: PascalCase in JSX (e.g., `CompassDemo.jsx`, `ProfileWizard`)
- Utilities: camelCase (e.g., `ucsbApi.js`, `useDatabase.js`)
- Tests: `.test.js` or `.spec.js` suffix
- Scripts: kebab-case or camelCase (e.g., `import-courses.js`, `transcriptparser.py`)

**Directories:**
- Lowercase with optional hyphens (e.g., `src/data`, `src/utils`, `src/hooks`)

**Variables & Functions:**
- camelCase for functions, variables: `extractCourseId()`, `fetchCourses()`
- PascalCase for React components and classes: `ProfileWizard`, `DashboardView`
- UPPER_CASE for constants: `UCSB_API_KEY`, `PROXY_BASE_URL`

**Database:**
- Tables: snake_case (e.g., `courses`, `course_prerequisites`, `requirement_categories`)
- Columns: snake_case (e.g., `course_id_clean`, `dept_code`, `units_fixed`)

## Where to Add New Code

**New Feature:**
- Primary code: `CompassDemo.jsx` (add component or helper function)
- Tests: `tests/components/` or `tests/utils/`
- Hooks: Add to `src/hooks/useDatabase.js` if querying Supabase
- Styles: Inline styles in JSX (current pattern) or add to `src/styles/` if extracting

**New Component/Module:**
- Reusable components: Create `.jsx` file in `src/components/` (currently empty)
- Export as named export or default
- Import in `CompassDemo.jsx` or other components

**Utilities:**
- API wrappers: Add to `src/utils/ucsbApi.js` or create new file in `src/utils/`
- Data helpers: Add to `src/hooks/useDatabase.js` or create new utility
- Formatters/validators: Create in `src/utils/` with descriptive name

**Database Changes:**
- New migrations: Create `src/data/migrations/002_feature_name.sql`
- Follow UUID generation and timestamp patterns from `001_initial_schema.sql`
- Include indexes for foreign keys and frequently queried columns

**Data Pipeline:**
- New import script: Add to `scripts/` directory
- Follow async/await pattern used in existing scripts
- Use Supabase client from `src/lib/supabase.js`

## Monolithic Component Warning

**CompassDemo.jsx (3991 lines)** contains:
- All UI views (8 major components)
- All state management
- All business logic (course filtering, requirement validation, etc.)
- All utility functions (extractCourseId, buildUserRequirements, etc.)
- Mock data (requirements, quarter plans, known courses)
- Theme and styling (inline CSS objects)

**Refactoring opportunity:** Extract components and utilities:
1. Components → `src/components/` (ProfileWizard, DashboardView, etc.)
2. Hooks → `src/hooks/` (useEconBARequirements, etc.)
3. Utils → `src/utils/` (extractCourseId, buildUserRequirements, etc.)
4. Data → `src/data/mock.js` (requirements, quarterPlan, knownCourses)
5. Styles → `src/styles/theme.js` (theme object)

Current structure works for demo/MVP but will need refactoring for production scale.
