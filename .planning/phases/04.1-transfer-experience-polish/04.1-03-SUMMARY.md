# Phase 4.1 Plan 03: Major Card CC Course Pairing Display Summary

Major Card (Card 3) in TransferDashboardView now shows the satisfying CC course code (e.g., "via MATH 3A") inline next to each satisfied UCSB requirement, matching the display quality already present in TransferRoadmapView.

## What Was Built

The Card 3 major lower-division requirements list in `TransferDashboardView.jsx` was updated to:

1. Perform an articulation lookup for each satisfied UCSB course ID using `articulations.find()` with `articulation_type === 'equivalent'`, `target_course.course_id_clean === courseId`, and `completedCcCourses.includes(art.source_course_code)`
2. Display "via COURSE_CODE" in muted gray below the "Satisfied" status label when a match is found
3. Continue showing "Satisfied" (no code) when isSatisfied but no exact articulation match exists
4. Continue showing "Not satisfied" in gray for unmet requirements

## Decisions Made

- **Lookup convention:** `articulation_type === 'equivalent'` only — matches Phase 03 convention. Combined/partial articulations excluded by design (same as `mapCcCoursesToUcsbRequirements`).
- **Graceful fallback:** When `isSatisfied` is true but no exact articulation row found, shows "Satisfied" without a course code. Prevents blank/broken display on stale articulation data.
- **Layout change:** `alignItems: 'center'` changed to `'flex-start'` to handle multi-line right side (status + code stacked vertically). `gap: theme.spacing[2]` added.
- **No raw hex strings:** All colors use `theme.colors.*` — passes existing theme-usage test enforcement.
